query_data()

query_data(integer  $courseid, integer  $year, integer  $coursestartweek, integer  $currentweek, integer  $userid = null, boolean  $ignoreweeks = false, boolean  $onlyunviewed = false) : array

Queries the needed data for making associations. It queries the number of accesses a user makes to a module. This function is used both for historic and current data (determined by the parameters).

The needed data is the following: moduleid: the id of the module (resource) that has been accessed. module_name: the name of the module (resource) that has been accessed. userid: the id of the user who has accessed the module. log_views: the number of times the user has accessed the module.

Parameters

integer $courseid

The id of the course from which the data will be extracted.
integer $year

The year the course is/was teached.
integer $coursestartweek

The start week of the course.
integer $currentweek

The week until the data will be queried. For a week n, the queried data will be from the first week, to the week (n-1). When querying historic data, this week will (presumably) be the end week of the course.
integer $userid

The user id to calculate the views of. Default is null, so, by default the logviews are queried for all the users of the course.
boolean $ignoreweeks

If the weeks have to be ignored. By default is false, so, by default the weeks are taken into consideration
boolean $onlyunviewed

If only the unviewed resources have to be queried. By default is false, so, by default all the resources are queried, viewed or not.

Returns

array —

An object for each record in recordset.

Tags

insert_associations()

insert_associations(integer  $number, array  $currentusersids, integer  $currentcourseid, array  $historicuserids, integer  $historiccourseid, integer  $week)

Inserts the calculated associations between students for a specific course, into the associations table. Instead of receiving an associative array with all the fields, an array of each field is received, for the encapsulation of the data logic in the class (in this case, the names of the columns). A little more work at the creation of the function, but for a proper encapsulation that results in an easier maintenance.

Obviously, arrays' values' indexes will be aligned.

Parameters

The number of associations to insert.

integer $currentcourseid

The id of the course the current users belong to. As the associations are calculed for a course, it's a single int value.

The ids of the historics users, which are associated to current users.

The id of the course the historics users belong to. As the associations are calculed for a course, it's a single int value.

insert_selections(array  $selections, integer  $courseid, integer  $year)

start_transaction()

start_transaction() : object

Initiates a database transaction. The changes made after calling this function will not be saved until commit_transaction is called (with the returned object passed as argument), so, this must be used carefully.

The main purpose of this is to try a set of instructions (inserts into different tables, usually) that fulfill the defined integrity rules, to commit the operations if they fulfill, or to rollback and not save any change if they don't and an exception has been thrown. In short, this is though to be used within a try/catch block, where the last instruction of the try would be the commit_transaction call; and the instruction of the catch would be the rollback_transaction call.

Returns

object —

Transaction object.
